# IDE Integration Guide — 5 AI Coding Agents

Armature integrates with 5 AI coding agents, providing quality gates, architectural enforcement, budget tracking, and self-healing across every IDE. Each integration is generated from your single `armature.yaml` config.

```bash
# Generate all integrations at once
armature hooks --all

# Or generate individually
armature hooks --claude-code
armature hooks --cursor
armature hooks --copilot
armature hooks --windsurf
armature hooks --github-actions
```

---

## 1. Claude Code Integration

**What it generates**: `.claude/settings.local.json`

Claude Code has the deepest integration — Armature wires into Claude's hook system for real-time quality enforcement.

### Hooks Installed

| Hook | Trigger | What It Does |
|------|---------|--------------|
| `PostToolUse` | Every `Write` or `Edit` tool call | Runs `armature check --file <path>` — instant lint + type feedback |
| `PreSession` | Session start | Runs `armature pre-dev --env-check-only` — validates tools installed |

### Permissions Auto-Configured

```json
{
  "permissions": {
    "allow": [
      "Bash(armature check *)",
      "Bash(armature heal *)",
      "Bash(armature gc *)",
      "Bash(armature budget *)",
      "Bash(armature baseline *)",
      "Bash(armature pre-dev *)",
      "Bash(armature post-dev *)"
    ]
  }
}
```

### MCP Server (Direct Tool Access)

Claude Code can also call Armature tools directly via MCP:

```json
// .claude/settings.local.json
{
  "mcpServers": {
    "armature": {
      "command": "python",
      "args": ["-m", "armature.mcp.server"]
    }
  }
}
```

This exposes 13 tools directly in Claude's tool palette:

| MCP Tool | Purpose |
|----------|---------|
| `armature_check` | Run quality gates (8 weighted checks) |
| `armature_heal` | Self-healing pipeline (lint auto-fix, type/test reporting) |
| `armature_gc` | Garbage collection (dead code, stale docs, drift) |
| `armature_budget` | Log token/cost usage per spec |
| `armature_preplan` | Pre-development budget estimation |
| `armature_benchmark` | Industry benchmark comparison |
| `armature_estimate` | Token/cost estimation for a task |
| `armature_baseline` | Capture quality baseline snapshot |
| `armature_pre_dev` | Pre-development environment check |
| `armature_post_dev` | Post-development regression detection |
| `armature_route` | Multi-provider model routing decision |
| `armature_calibrate` | Auto-calibrate budget from actuals |
| `armature_cache_stats` | Semantic cache hit/miss statistics |

### Example Workflow in Claude Code

```
You: "Add user authentication to the login page"

Claude Code:
  1. [PreSession hook] → armature pre-dev validates ruff, mypy, pytest installed ✓
  2. Claude writes auth.py
  3. [PostToolUse hook] → armature check --file auth.py
     → lint: clean ✓
     → type_check: 2 errors (missing return type)
  4. Claude fixes type annotations
  5. [PostToolUse hook] → armature check --file auth.py → clean ✓
  6. You: "run full quality check"
  7. [MCP] armature_check → 8/8 checks pass, score: 0.94 (review_ready)
  8. [MCP] armature_heal → auto-fixes remaining lint warning
  9. [MCP] armature_check → score: 0.97 (merge_ready) ✓
```

### Setup

```bash
# One command
armature hooks --claude-code

# Or enable in armature.yaml
integrations:
  claude_code:
    enabled: true
    post_tool_use: true
    pre_session: true
```

## 2. Cursor Integration

**What it generates**: `.cursor/rules`

Cursor uses a rules file to guide its AI assistant. Armature generates architecture constraints and quality requirements.

### Generated Rules Content

```markdown
# Armature -- Architectural constraints for Cursor
# Generated by: armature hooks --cursor

## Layer Boundaries

- Code in `cli` layer MUST NOT import from: budget, quality, heal
- Code in `quality` layer MUST NOT import from: cli, mcp
- Code in `budget` layer MUST NOT import from: cli, mcp, quality

## Shared Infrastructure (always importable)

- `_internal`
- `config`

## Class Conformance

- Classes matching `*Healer*` must inherit from `BaseHealer`
  Required methods: heal, can_fix

## Quality Requirements

- Run `armature check` before committing
- Quality gates: draft=0.7, review_ready=0.85, merge_ready=0.95
```

### MCP Server for Cursor

```json
// .cursor/mcp.json
{
  "mcpServers": {
    "armature": {
      "command": "python",
      "args": ["-m", "armature.mcp.server"],
      "env": {
        "ARMATURE_PROJECT_DIR": "."
      }
    }
  }
}
```

### Setup

```bash
armature hooks --cursor

# Or enable in armature.yaml
integrations:
  cursor:
    enabled: true
```

## 5. GitHub Actions (CI/CD)

**What it generates**: `.github/workflows/armature.yml`

Armature runs as a CI quality gate on every push and PR.

### Generated Workflow

```yaml
# Armature Harness Engineering CI
# Generated by: armature hooks --github-actions

name: Armature

on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

jobs:
  armature-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install dependencies
        run: |
          pip install armature-harness[python]
          pip install -e ".[dev]" 2>/dev/null || true

      - name: Armature Check
        run: armature check --json

      - name: Armature GC
        run: armature gc --json
        if: github.event_name == 'pull_request'
```

### Setup

```bash
armature hooks --github-actions

# Or enable in armature.yaml
integrations:
  github_actions:
    enabled: true
```
